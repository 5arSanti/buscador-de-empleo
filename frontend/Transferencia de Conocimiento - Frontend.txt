Teniendo en cuenta que el codigo fuente del Frontend de la BUE se va a encontrar en las carpetas de Desarrollo y Pruebas del Servidor, es necesario realizar los cambios de direccionamiento de visualizacion de archivos estaticos (index.html)

Para generar una build del Frontend, se debe dirigir a la carpeta raiz de la carpeta frontend en el codigo fuente y ejecutar el siguiente comando:

	npm run build

Una vez ejecutado este comando, se crear√° una carpeta dist encargada de contener los archivos necesarios de la Build construida listos para ser desplegados en PRUEBAS, por lo tanto el direccionamiento de archivos estaticos debe terminar en /dist/.


En el archivo
	app.properties.env

Ubicado en la raiz de la carpeta frontend se encuentran los datos del dominio de conexion al backend, y la estructura de la API, 
la cual debe ser igual a la estructura de la API del servidor backend.

Es importante que al realizar un cambio en este archivo desde el codigo fuente, es necesario ejecutar el comando "npm run build" para generar la version con los nuevos datos de conexion y que se vizaulice correctamente en PRUEBAS y DESARROLLO.
